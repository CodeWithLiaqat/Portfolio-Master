## 3 · WEBSITE & TECHNICAL ARCHITECTURE

**Stack — use exactly this, for these reasons:**

| Layer | Choice | Why |
|---|---|---|
| Framework | **React 19 + TypeScript (strict)** | Concurrent features for canvas + DOM coexistence |
| Routing/rendering | **React Router 7, framework mode, `prerender` for all static routes** | Real HTML for crawlers and instant LCP despite a WebGL-heavy SPA. This resolves the SEO-vs-canvas conflict — never render meaningful content inside the canvas |
| Styling | **Tailwind CSS 4 + CSS custom properties for all tokens** | Themes switch by swapping one `data-theme` attribute |
| Scroll engine | **GSAP + ScrollTrigger + SplitText**, smoothing via **Lenis** | The industry-proof pairing for scrubbed, pinned cinematic scroll |
| UI micro-interactions | **Motion (motion.dev — the successor to Framer Motion)** | Layout animations, presence, gestures on UI chrome only |
| 3D | **Three.js via React Three Fiber + drei + @react-three/postprocessing** | Declarative scenes, shared React state with the DOM |
| Page transitions | **View Transitions API**, GSAP overlay fallback | Native, GPU-composited, 2026 best practice |
| Utility UI | **shadcn/ui** for Dialog, Accordion, Toast, form primitives — fully restyled to the tokens in §6. It must be unrecognizable as shadcn | Accessibility for free without a template look. Do not use stock 21st.dev blocks for any hero, nav, card, or signature moment — those are custom |
| Content | **MDX** for Insights & Resources, typed frontmatter | Editorial content without a CMS |
| Backend | Minimal **Express** server (contact endpoint via Resend or SMTP) serving the prerendered build | One deployable Replit app |

**Hard architectural rules:**
- One single fixed, full-viewport `<Canvas>` mounted at the app root behind the DOM (`pointer-events: none` except where a scene opts in). Scenes mount/unmount per route; scroll drives them via a shared progress store (Zustand), never via drei `ScrollControls`.
- Lenis is the single scroll source: wire `lenis.raf` into `gsap.ticker`, and `ScrollTrigger.update` to Lenis scroll events. Native scrolling is never hijacked — no wheel-event takeover, no forced scroll positions.
- The entire 3D bundle (three, R3F, drei, scenes) is one lazy chunk, loaded during the preloader. The DOM hero renders and is interactive even if the canvas is still loading.

## 6 · UI DESIGN SYSTEM

**Spacing:** 4px base unit. Component paddings from the 4/8/12/16/24/32/48/64 ladder. Section vertical padding: `clamp(96px, 12vw, 200px)`. Container: `max-width: 1440px`, gutters `clamp(20px, 5vw, 96px)`. Grid: 12 columns desktop / 6 tablet / 4 mobile.

**Radii:** 8px inputs · 16px cards · 24px modals · 999px pills. Never mix radii on sibling elements.

**Glass system (one recipe, reused):** `background: color-mix(in oklab, var(--surface) 65%, transparent); backdrop-filter: blur(20px) saturate(140%); border: 1px solid color-mix(in oklab, var(--accent) 18%, transparent)` plus a 1px inner top highlight. Glass is reserved for floating chrome (nav, theme controls, toasts) — not slathered on every card.

**Glow system:** glow is *light behaving physically*, never a drop-shadow decoration. Implemented as radial-gradient pseudo-elements in the accent color at 10–25% opacity, blurred 40–80px, positioned as if cast by the nearest light source. Interactive glow follows the cursor on CTAs and the contact scene.

**Shadow/elevation (3 levels only):** e1 `0 2px 8px rgb(0 0 0 / .35)` · e2 `0 12px 32px rgb(0 0 0 / .45)` · e3 `0 32px 80px rgb(0 0 0 / .55)`. Dark UIs read depth from light edges as much as shadow — every elevated surface gets a subtle 1px top border in `--text` at 6% opacity.

**Buttons:** Primary — pill, accent-filled, ink-dark label, magnetic (§9), cursor-tracking sheen; Secondary — 1px accent-tinted border, transparent, fills on hover via a clipped radial reveal from the cursor point; Ghost — text + animated underline that draws left→right; Icon — 44px min target. States for hover/active/focus-visible (2px offset outline in accent)/loading (label morphs to inline spinner via Motion layout) are mandatory on all variants.

**Cards:** never flat. The house card = deep surface (`--surface-2`), e2 elevation, 3D tilt on pointer (max 6°, spring-smoothed), a specular highlight that tracks the cursor, and content that parallaxes 4–8px against the card body. Case-study cards add the WebGL hover displacement (§11).

**Forms:** shadcn primitives restyled — floating labels, 1px bottom-border fields that ignite in accent on focus, inline validation appearing via Motion presence, and an error style that explains the fix ("Enter an email like name@studio.com").

**Iconography:** one set only — Lucide, 1.5px stroke, sized 20/24. Icons may animate (draw-on, morph) but never bounce.

**Backgrounds:** every section sits on the living background system — base dark gradient per theme + animated mesh-gradient blobs (two, slow, ±60s loops, GPU-composited) + 2.5% film grain (tiling noise, `mix-blend-mode: overlay`) + occasional volumetric light ray on signature sections. Nothing is ever flat black.

**Semantic (theme-independent):** success `#5FBF8A` · warning `#E5B461` · error `#E0655F` · borders `color-mix(in oklab, var(--text) 10%, transparent)`.

**Theme behavior — implement exactly this logic:**
1. Default mode is **Auto-Cinematic**: on Home, Acts I–II run Aurum, Acts III–V cross-fade to Iris, Acts VI–VII to Garnet, scrubbed over ~15% of scroll between acts. The 3D lighting, mesh gradients, and all tokens transition together (interpolate tokens via GSAP on the root element; the canvas reads the same store).
2. Three floating theme controls (small glass orbs filled with each theme's gradient, bottom-right, tooltip-labeled) let the user choose. **A manual choice pins the theme** — auto transitions stop, and scroll instead modulates *lighting intensity and gradient position within* the pinned theme, so the cinema never dies. A fourth subtle "auto" control restores cinematic mode. Persist the choice in localStorage.
3. Theme transitions take 1.2s with the `cinematic` ease, and the Artifact's refracted light visibly changes color mid-transition — the theme switch is itself a wow moment.

## 9 · MOTION DESIGN BIBLE

**The grammar (every animation on the site obeys this):**
- **Physics of the world:** everything behaves like heavy glass in low gravity — decisive starts, long elegant settles, no bounce, no elastic, no wobble.
- **Easing tokens:** `--ease-out` `cubic-bezier(0.22, 1, 0.36, 1)` (expo-out — 90% of entrances) · `--ease-inout` `cubic-bezier(0.83, 0, 0.17, 1)` (position changes, theme shifts) · `cinematic` — a GSAP CustomEase `M0,0 C0.2,0 0.15,1 1,1` for hero/act moments. Linear only inside scrubbed ScrollTriggers (the scrub itself supplies the feel).
- **Duration tokens:** micro 0.25s · standard 0.7s · cinematic 1.4s · page transition 0.9s. Stagger: 0.06s UI / 0.1s content / 0.02s per SplitText line-char.
- **Choreography law:** one focal animation at a time per viewport; supporting elements follow it, never compete. Entrances animate `opacity`, `transform`, `clip-path` only (compositor-safe). Exit animations are half the duration of entrances.
- **Distance law:** reveal translations are 24–40px — perceptible, never a 200px flight.

**The vocabulary — seven expressions of the one grammar, mapped to Home's seven acts so "every section introduces a new style" stays coherent:** Ⅰ materialization (particles→form) · Ⅱ pinned chapter scrub · Ⅲ sticky-stack morph · Ⅳ horizontal gallery drift · Ⅴ counter/constellation assembly · Ⅵ editorial mask reveals · Ⅶ convergence & light. Inner pages reuse these expressions; they never invent an eighth ad hoc.

**Signature motion set:**
- **Headlines:** SplitText into lines, each line inside an `overflow:hidden` mask, rising 110%→0 with 0.08s stagger, `--ease-out`. This is the house reveal — used everywhere, so it becomes the brand's motion signature.
- **Images:** reveal via `clip-path: inset()` wipe in the scroll direction + 1.06→1 scale settle.
- **Magnetic buttons:** within a 120px radius the button translates up to 12px toward the cursor and its label up to 4px more (two-layer parallax), spring `stiffness 150 / damping 15` via Motion; releases to origin on leave.
- **Cursor (fine pointers only):** 8px dot + 32px trailing ring (0.12 lerp). Contextual morphs: ring expands with "View" label over case studies; becomes a vertical-drag glyph over the gallery; tints to accent over CTAs; hides over the Artifact where the object itself reacts instead. Fully removed on touch devices and under reduced motion.
- **Nav:** glass bar that condenses on scroll-down, returns on scroll-up. Menu button opens a full-screen overlay: background veil wipes in (`--ease-inout`), links reveal via house headline treatment; hovering a link shifts a large ghosted preview word behind the list. Progress line under the bar fills in the live theme gradient.
- **Page transitions:** View Transitions API — shared-element morph for case-study images (card image → hero image), root cross-fade 0.9s elsewhere; GSAP theme-gradient veil wipe as fallback for unsupported browsers.
- **Preloader (the overture, and the 3D chunk's loading time made beautiful):** black screen, mono counter 000→100 tracking real asset progress, the Artifact's particles drifting inward and assembling as it climbs; at 100 the counter cuts, particles snap into the Artifact with a single light bloom, veil lifts into the hero. Max 2.5s even on fast connections (minimum dwell 1.2s so it never flickers). Skippable via click/keypress. Shown once per session.
- **Micro-interactions inventory (all mandatory):** link underline draw-ins · icon draw-on at first reveal · accordion chevrons rotating with height-animated panels (Motion layout) · form focus border ignition · toast slide+fade · tab indicator sliding via Motion `layoutId` · tooltip 0.25s fade+2px rise · stat counters counting only when ≥50% in view, mono, once.

**GSAP architecture (build it this way):**
- `src/motion/` exports: easing/duration tokens, `revealLines()`, `revealImage()`, `pinAct()`, `counter()` utilities; all components consume these — no inline magic numbers.
- Every component uses `useGSAP` (official React hook) with proper context cleanup; ScrollTriggers are created after Lenis mounts and refreshed on route change and font load.
- `gsap.matchMedia()` gates: `(min-width:1024px)` full choreography · `(max-width:1023px)` simplified (no pinning except Act Ⅲ, shorter distances) · `(prefers-reduced-motion: reduce)` sets the global static mode.
- Pinned sections use `pinSpacing: true`, scrub `0.8–1.2` (never `true`/0 — the slight lag is the cinematic feel), and `anticipatePin: 1`.

**GSAP vs Motion responsibilities (strict):** GSAP = everything scroll-linked, SplitText, pinning, the theme interpolation, canvas-progress driving. Motion = presence (mount/unmount), layout animations (tabs, accordions, form states), gesture springs (magnetic, tilt). If both could do it, scroll-linked → GSAP, state-linked → Motion.

**Mobile motion:** no cursor system, tilt from `deviceorientation` disabled by default (battery), pinning minimized, Artifact drops post-processing and halves particle counts, all hover effects have tap equivalents or graceful absence. Quality stays premium via typography, spacing, and the (cheaper) canvas — never via more effects.

**Reduced motion (a designed mode, not a kill switch):** scrubbed sequences become static compositions at their end state; the canvas renders a single lit still of the Artifact per act (no rotation); reveals become 0.3s opacity fades; parallax, magnetic, cursor, auto theme-cinema all off (manual theme switching stays, with instant swaps); preloader becomes a simple fade. The site must remain fully beautiful and fully usable in this mode — test it deliberately.

## 11 · THREE.JS / R3F SCENE PLAN

**The Artifact — identity:** a monolithic form composed of 9–13 interlocking faceted glass shards (custom low-poly geometry, ~15–25k triangles total — sculpted, not a primitive; no torus knots, no icosahedron clichés). Material: drei `MeshTransmissionMaterial` (transmission 1, thickness ~1.2, chromatic aberration ~0.04, roughness ~0.12) with an inner emissive core mesh supplying the theme-colored light that refracts outward. This one object, restated, is the entire 3D identity of the site — no unrelated 3D anywhere.

**Scene architecture:** single root `<Canvas>` (fixed, `dpr={[1, 1.75]}`, `gl={{ antialias: true, powerPreference: 'high-performance' }}`); per-route scene components; a Zustand store `{ act, actProgress, theme, pointer }` written by GSAP ScrollTriggers and read inside `useFrame` with damped interpolation (`maath/easing` `damp3`/`dampE`, λ≈4) so canvas motion is always smoothed, never raw scroll.

**Lighting philosophy:** cinematic three-point — key directional in the theme accent, cool rim from behind for silhouette, faint fill; plus the core's point light. A subtle drei `Environment` (studio HDR at low intensity) for speculars. Lights lerp colors during theme transitions.

**Post-processing (desktop only):** Bloom (threshold ~0.85, gentle), Vignette, subtle Noise. No SSAO/DOF — too costly for the value. Mobile: no post-processing chain at all.

**Particles:** one instanced/points system (≤3,000 desktop / ≤1,200 mobile), additive blending, theme-colored, driven by a simple curl-noise shader; reused for preloader assembly, Act Ⅴ constellation, and Act Ⅶ convergence — one system, three roles.

**Custom shaders (small and purposeful):** gallery displacement/RGB-shift material for image planes; caustics-style background shader for Act Ⅵ; the mesh-gradient background may be a fullscreen shader plane if cheaper than CSS. No shader exists for its own sake.

**Performance rules:** `frameloop="demand"` + `invalidate()` whenever idle (Legal, FAQ, static states); geometry via draco if any assets are imported; textures ≤2048px, KTX2 where possible; `<AdaptiveDpr>` under load; dispose scenes on route unmount; the canvas never blocks LCP (DOM paints first, canvas fades in). Detect weak GPUs (`detect-gpu`): tier ≤1 gets the static-poster fallback with CSS-only ambiance — gracefully, with no error surface.
